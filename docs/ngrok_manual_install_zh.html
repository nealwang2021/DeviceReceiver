ngrok 手动安装与配置（Windows）

适用于本仓库 三轴台 Stage 桩 与 scripts/start_stage_with_ngrok.bat 联调场景； 以下为手动下载安装步骤（不做自动安装脚本）。

一、注册并获取 Authtoken

  1. 打开 https://ngrok.com，注册或登录。
  2. 进入 Dashboard（控制台），找到 Your Authtoken（或连接向导中的 token）。
  3. 复制整段 token（注意保密）。
安全：不要将 token 提交到 Git 仓库或发送给不可信方。

二、下载并放置 ngrok（Windows）

  1. 打开 https://ngrok.com/download，选择 Windows。
  2. 下载 zip，解压到固定目录，例如 C:\Tools\ngrok\，确保目录中有 ngrok.exe。

三、加入 PATH（任选一种）

方式 A（推荐）：用户级 PATH

  1. Win + R，输入 sysdm.cpl 回车。
  2. 「高级」→「环境变量」。
  3. 在「用户变量」中选中 Path →「编辑」→「新建」，填入 ngrok 所在目录，例如 C:\Tools\ngrok。
  4. 确定保存后，新开一个 cmd 或 PowerShell 窗口（旧窗口不会刷新 PATH）。

方式 B：不配置 PATH

每次使用 ngrok 的完整路径，例如：

C:\Tools\ngrok\ngrok.exe version

四、配置 Authtoken（一次性）

在新的 PowerShell 或 cmd 中执行（将 <你的TOKEN> 替换为控制台复制的 token）：

ngrok config add-authtoken <你的TOKEN>

配置会写入本机用户目录下的 ngrok 配置文件（具体路径以 ngrok 版本为准）。

验证：

ngrok version

能正常显示版本号即表示 CLI 可用；若已正确添加 authtoken，后续隧道才可正常使用。

五、与本项目联用（Stage 桩 + 外网地址）

  1. 进入本仓库根目录（例如 d:\WS\qtpro\DeviceReceiver）。
  2. 确认已安装 Python，且已执行 pip install grpcio protobuf。
  3. 在仓库根目录运行： 
    scripts\start_stage_with_ngrok.bat
    或指定端口： 
    scripts\start_stage_with_ngrok.bat -Port 50052
  4. 按控制台输出，将外网 gRPC 地址 host:port 填到远端 realtime_data 的 StageGrpcEndpoint（不要带 tcp:// 前缀）。
若提示找不到 ngrok：检查 PATH 是否生效，或使用 ngrok.exe 的完整路径；亦可查看 http://127.0.0.1:4040（隧道启动后）确认隧道信息。

六、防火墙（按需）

若连接异常，可在「Windows 安全中心 → 防火墙」中为 ngrok.exe、python.exe 允许专用网络访问（按需允许公用网络）。

七、常见问题

现象 处理
ngrok 不是内部或外部命令 PATH 未配置或未新开终端；或使用完整路径运行 ngrok.exe。
authtoken 无效 在控制台重新复制 token，避免多余空格或换行。
脚本未打印外网地址 等待数秒后重试；浏览器打开 http://127.0.0.1:4040 查看；确认账号与 TCP 隧道策略（以 ngrok 官网为准）。
无法运行 .ps1 仅当前用户：Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

八、安全提醒

当前 Stage gRPC 为 明文 insecure，仅适合测试；经 ngrok 中继的流量请勿用于不可信环境或长期公网暴露。生产环境请使用 TLS、VPN 或专线。
