# 自愈修复技能

## 目标

在不突破架构约束的前提下，为常见工程失败提供可重复、可收敛的自动修复流程。

## 触发条件

出现以下任一情况时触发：

- lint/type/build/test 命令失败。
- 代码改动后程序启动失败。
- 验证步骤发现当前任务引入回归。

## 修复流程（最多 3 轮）

### 第 1 轮：局部修复

1. 识别最小失败范围（单文件/符号/配置）。
2. 施加最小且安全的修复。
3. 仅重跑原失败命令。

### 第 2 轮：根因修复

1. 根据下表判定错误类别。
2. 修复根因，避免仅做表面补丁。
3. 重跑失败命令 + 一条相邻防护命令。

### 第 3 轮：稳定化

1. 增加防护（断言/检查/测试/配置加固）防止复发。
2. 重跑相关验证集。
3. 若仍失败，停止修复并输出阻塞报告。

## 错误分类策略

| 类别 | 典型信号 | 首选动作 | 升级路径 |
|---|---|---|---|
| 语法/编译 | 编译器语法或类型报错 | 修复局部语法/类型不匹配 | 检查 include 顺序、宏、工具链参数 |
| 链接/构建配置 | 未解析符号、目标不匹配 | 核对 CMake 目标链接与选项 | 审核平台特定库与构建参数 |
| 测试回归 | 断言失败、行为漂移 | 用最小用例复现 | 对照预期行为与最近改动 |
| 线程并发 | 死锁、竞态、UI 卡死 | 隔离线程边界与共享状态 | 强制采用信号槽/线程安全队列模型 |
| 持久化兼容 | 旧文件无法加载或崩溃 | 增加兼容解析保护/回退 | 增加版本处理与迁移路径 |
| 环境/工具 | 工具缺失、路径或配置错误 | 校验环境前提与命令假设 | 记录前置条件或备用命令路径 |

## 阻塞报告格式

3 轮内无法修复时，按以下格式输出：

```text
[SELF-HEAL BLOCKED]
Task: 
Failure Class: 
Failing Command: 
Observed Error: 
What Was Tried (Round 1-3): 
Current Risk: 
Recommended Next Action: 
Owner Input Needed: 
```
