# 架构决策记录（ADR）

用于记录重要技术决策。

---

## 决策标题

## 状态

- Proposed / Accepted / Superseded / Rejected

## 日期

- YYYY-MM-DD

## 背景

- 这个决策要解决什么问题或约束？
- 影响范围与紧急程度是什么？

## 备选方案

1. 方案 A
2. 方案 B
3. 方案 C

## 权衡分析

- 方案 A：优点 / 缺点
- 方案 B：优点 / 缺点
- 方案 C：优点 / 缺点

## 结论

- 选择了哪个方案，理由是什么。

## 影响

- 正向影响
- 负向影响
- 运行与维护影响

## 兼容性与风险

- 向后兼容影响
- 线程/性能/安全风险
- 缓解措施

## 回滚方案

- 若方案失败，如何安全回退。
